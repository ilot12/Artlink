# ArtLink DB 사용 방법 (팀원용)

이 문서는 이미 생성된 ArtLink 데이터베이스에
팀원이 **백엔드 서버를 연결하여 사용하는 방법**을 설명합니다.

DB 자체를 생성할 필요는 없으며,
.env 설정만 하면 바로 사용 가능합니다.

동주 주석
Prisma를 이용해서 Supabase에 DB 스키마를 바꿀 때는 요청하는 URL인 DATABASE_URL 말고 DIRECT_URL이 필요합니다.
현재 제가 Supabase에 세팅을 해 두어서 요청만 하고 스키마를 바꾸는게 아니라면 필요는 없습니다.
임의로 기능을 추가/수정하고 싶으시면 Setup txt에 직접 Supabase를 기반으로 DB를 구축하는 법을 설명해두었습니다.

---

1. 저장소 clone

---

프로젝트를 먼저 clone 합니다.

git clone <repository-url>

이후 backend 폴더로 이동합니다.

cd backend

---

2. 패키지 설치

---

필요한 패키지를 설치합니다.

npm install

---

3. .env 파일 생성

---

backend 폴더에 `.env` 파일을 생성합니다.

예시

DATABASE_URL="postgresql://postgres.xxx:PASSWORD@aws-1-ap-northeast-1.pooler.supabase.com:6543/postgres?pgbouncer=true&connection_limit=1"

JWT_SECRET="your-secret-key"

PORT=4000
NODE_ENV=development
FRONTEND_URL="http://localhost:5173"

UPLOAD_DIR="uploads"

설명

DATABASE_URL
→ Supabase Pooler를 이용한 DB 연결 주소

pgbouncer=true
→ Supabase connection pooler 사용

connection_limit=1
→ 서버에서 사용하는 DB 연결 수 제한

---

4. 서버 실행

---

서버를 실행합니다.

npx tsx src/index.ts

또는

npm run dev

실행되면 다음 메시지가 출력됩니다.

🚀 ArtLink 백엔드 서버 실행 중: http://localhost:4000

---

5. DB 연결 확인

---

다음 API를 요청하여 확인합니다.

GET
http://localhost:4000/api/galleries

데이터가 반환되면 정상적으로 DB 연결이 완료된 것입니다.

---

6. 현재 시스템 구조

---

Frontend
↓
Backend API (Node Server)
↓
Supabase PostgreSQL

프론트엔드는 백엔드를 통해서만 DB에 접근합니다.

---

7. 주의사항

---

.env 파일은 절대 Git에 올리지 않습니다.

.gitignore에 반드시 포함되어 있어야 합니다.

.env에는 DB 비밀번호와 같은 민감 정보가 포함되어 있습니다.
