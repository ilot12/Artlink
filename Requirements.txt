이 문서는 ArtLink 플랫폼의 기술적 청사진입니다. 개발자나 AI 에이전트가 시스템을 100% 충실하게 재구성할 수 있도록 상세하게 설계되었습니다.

요약 : 작가와 갤러리를 이어주는 모바일 웹 (PWA 지원)

[사용 기술 Stack]

프레임워크 : React / Vite / TypeScript

라우팅 : React-router v6

데이터 요청 : Axios

서버 상태 관리 : TanStack Query v5

클라이언트 및 전역 상태 관리 : Zustand v5

스타일링 : Shadcn/ui, Tailwind CSS, Framer Motion

폼 관리 및 검증 : React-hook-form, Zod

코드 퀄리티 : ESLint, Prettier, TypeScript

테스트 : Vitest

백엔드 : Express

DB : PostgreSQL + 프리스마

인증 : Auth.js

유저 타입 : Admin / Gallery / Artist

[유저 시나리오]

1. 초기 접속시
    1. SplashScreen을 통해 화면 가운데에 ArtLink logo 표기
    2. 세련되고도 깔끔한 Animation을 적용
    3. 1초간 유지
2. Main 화면
    1. Hero Section : 상단
        1. 유저가 slide 가능하게 해야함
        2. 신규 등록 및 삭제, 수정 권한은 Admin user에게만 있음.
            1. 사진 / 제목 / 링크 URL 등록 가능한 기능을 만들어야하고, 이 등록 기능은 Admin user만 보이며 역시 admin user만 사용 가능하게 해야함
            2. 링크 URL 등록된 경우 [바로가기] 버튼을 우측 하단에 두어 페이지 이동되도록 함
                1. 외부 URL인 경우 웹브라우저를 통해 해당 사이트로 이동
                2. 우리 웹 내 페이지로 이동할 경우 해당 페이지로 이동
        3. 한 사진당 3초 유지 후 자동으로 다음 사진으로 넘어가도록 하고, 사용자가 슬라이드하여 사진 변경했을 경우 해당시점으로부터 3초 뒤 다시 자동 슬라이드 시작되도록 함.
    2. Center Catchphrase : 중단
        1. “갤러리와 아티스트를 잇다 : ArtLink” 이 문구를 세련되고 감각적으로 Center Catchphrase로 사용
    3. Quick Action Cards : Center Catchphrase 밑
        1. 3개의 각기 다른 페이지로 이동할 수 있도록 함 (갤러리 찾기 / 진행중인 공고 / 혜택)
    4. Navigation Bar
        1. 우측 상단에 홈 / 갤러리 / 모집공고 / 혜택 / 마이페이지 라는 이름의 페이지 이동 버튼 생성
        2. 좌측 상단에 ArtLink라는 로고 생성, 클릭시 홈화면 이동
        3. 모든 다른 페이지에서도 보여야함
    5. Gallery of the Month
        1. 이달의 갤러리로 선정된 갤러리들을 보여줌
            1. 갤러리가 등록한 메인 사진, 주소, 한줄 소개, 별점을 보여줌
            2. 해당 갤러리 클릭시 갤러리 상세 페이지로 이동
        2. 등록 및 관리는 admin 계정에서만 할 수 있음
3. 마이페이지
    1. 로그인되어있지 않을 시 로그인 창을 띄움 / 로그인되어있을 시 로그아웃 창 누르면 로그아웃되고 로그인창으로 이동
    2. 우선 개발용 퀵 로그인으로 Aritst 1 / Artist 2 / Gallery / Admin 계정 중 선택하여 로그인할 수 있는 창을 띄우도록 함
    3. 추후 실제 로그인 (메일 and 소셜 로그인) 기능으로 변경 가능하도록 설계
    4. 로그인 되었을 경우
        1. 프로필 사진 변경 기능
        2. 유저 type별 기능
            1. artist user
                1. 포트폴리오 작성 및 저장 기능
                    1. 전시 참가 이력 입력 및 수정 및 저장 기능
                    2. 작가 약력 입력 및 수정 및 저장 기능
                    3. 작품 사진 등록 및 저장 기능 / 등록한 사진 삭제 기능 / 최대 30개까지 등록 가능
                2. 등록되어있는 갤러리 및 공모에 대해 찜한 목록 모아보기 기능 / 찜하기 취소시 목록에서 빠져야함
                    1. 갤러리는 갤러리명이 뜨게, 공모는 갤러리명-공모명 으로 뜨게.
                    2. 갤러리 / 공모 각각 따로 볼 수 있게해야함
                3. 내가 작성한 리뷰 모아보기 기능
                4. 내가 지원한 공고 모아보기 기능
            2. 갤러리 user
                1. 자기가 운영하는 갤러리를 등록하는 요청을 보내는 기능
                    1. 요청시 갤러리 이름 / 주소 / 소개 / 대표자명 / 전화번호 / 대표 image / 검색시 사용될 지역 tag 선택 후 등록 요청할 수 있도록 해야함
                    2. 등록 요청 후 해당 요청의 status (승인 대기중 / 승인 거절) 확인할 수 있게해야함 / 승인되었을경우는 바로 본인 갤러리 목록에 보이게.
                    3. 승인은 admin이 해야하며 admin의 마이페이지에 이를 승인 /거절하는 이유 만들어야함 (거절시 사유작성)
                2. 공모를 등록하는 요청 보내는 기능
                    1. 등록시 본인 갤러리 중 하나 선택할수있게 / 공모 제목 / 개인전,아트페어 선택/ 공모 마감일 / 전시일 / 모집인원 / 전시지역 / 간단소개 를 작성해 요청해야함.
                    2. 승인은 admin이 해야하며 admin의 마이페이지에 이를 승인 /거절하는 이유 만들어야함 (거절시 사유작성)
                3. 등록 승인 후에는 공모와 갤러리 상세 페이지가 생성되며 검색도 되지만, 상세 내용만 수정가능하며 등록요청시 적은 정보들은 수정불가. 수정시 admin에게 요청을 보내야함. 따라서 수정요청 기능도 있어야함.
            3. Admin user
                1. 갤러리 / 공모 등록 승인 queue 를 만들어 이를 승인 / 거절하는 기능
                2. 히어로 섹션 관리 기능
                    1. 현재 등록된 리스트 확인 및 수정 / 삭제 기능
                    2. 신규 등록기능 (사진 / 문구 추가 및 미리보기)
                    3. 추가시 Title / Description / Link(Optional) / 사진 등록
                3. 혜택 목록 관리 기능
                    1. 현재 등록된 리스트 확인 및 수정 / 삭제 기능
                    2. 신규 등록기능 (사진 / 문구 추가 및 미리보기)
                    3. 추가시 Title / Description / Link(Optional) / 사진 등록
                4. Gallery of the Month 관리 페이지
                    1. 갤러리를 검색하여 이달의 갤러리로 선정할 수 있게 하는 기능
                    2. 이미 등록되어있는 것 내리는 것 및 변경 가능 기능
                    3. 등록 기한을 설정하여 등록 기한이 지나면 자동으로 목록에서 제거되도록함
        3. 
4. 갤러리 찾기 화면
    1. 홈화면의 Quick Action Cards에서 갤러리 찾기 클릭시 이동 / 혹은 Navigation Bar에서 갤러리 클릭시 이동
    2. 현재 등록되어있는 갤러리들을 보여줌
        1. 한 줄에 한 개씩
        2. 사진 / 이름 / 주소 / 전화번호 / 한줄소개 / 찜하기버튼 / 별점 보여줌
    3. 필터 기능
        1. 큰 카테고리 : 지역 / 별점 - 클릭하지 않을시 지역 / 별점의 큰 카테고리만 보임
            1. 지역 클릭시 세부 지역 필터 적용 가능 (서울 / 경기 북부 / 경기 남부 / 대전 / 부산)
            2. 별점 클릭시 4점 이상 / 3점 이상 필터 적용 가능
            3. 세부 필터 클릭시 필터를 적용하여 갤러리 목록을 보여주고, 적용한 필터 목록은 화면에 보여줌
        2. 별점순으로 나열하기 기능 추가
5. 갤러리 상세 페이지
    1. 갤러리 찾기 화면에서 갤러리를 클릭해서 들어가거나, 메인화면의 Gallery of the Month 에서 갤러리 클릭해서 들어갈 수 있음. 혹은, 공모 상세 페이지에서 갤러리 정보 클릭해서 들어갈 수 있음.
    2. 상단에 갤러리가 등록한 사진들 , 스와이프 가능하게
        1. 우측 상단에 하트 표시로 찜하기 가능하게
    3. 그밑에 갤러리 이름 / 주소 / 별점 / 한줄소개 
    4. 그밑에 상세 소개 (해당 갤러리 주인인 갤러리 user가 등록 및 수정가능)
        1. 해당 갤러리 주인인 Gallery user인 경우 상세 소개란에 [수정] 버튼이 보이고 작동하게함
    5. 그 밑에 진행중인 모집공고 보여줌
        1. 모집공고명 / 아트페어인지 개인전인지 / 전시일 / 모집인원 / 지역 을 명시하고 모집마감 Dday 표기
    6. 마감되어 전시까지 끝난 공고의 경우 홍보용으로 해당 전시에서의 사진, 간단한 글을 해당 Gallery user가 쓸 수 있는 기능 추가 필요.
    7. 리뷰 및 평점기능
        1. 리뷰 등록할 수 있어야함 - artist user만 가능.
            1. 사진 (option) 및 글과 함께 평점 선택 후 등록
            2. 익명 선택 가능
                1. 익명 선택시 등록자와 admin user만 누가 쓴건지 확인 가능하게 함
                2. 익명으로 할시 ID가 익명의 예술가 1 /2 … 로 나오게 해줘
            3. admin user는 특정 댓글 삭제할 수 있도록함
                1. admin user일 경우 리뷰 삭제 버튼이 보이고 작동하도록함
6. 모집 공고 상세 페이지
    1. 홈화면의 진행중인 공고 클릭 / 혹은 Navigation Bar에서 모집공고 클릭하면 이동되도록함
    2. 현재 진행중인 공고들을 보여줌 (Dday 끝나지 않은 것만)
        1. 상세내용도 같이 보여줌 (사진, 제목, 갤러리명, 갤러리별점, 개인전인지 아트페어인지, 모집인원, 전시 지역, Dday)
        2. 갤러리명 쪽을 클릭하면 갤러리 상세페이지로 이동
        3. 찜하기 버튼
    3. 필터를 만들어 적용
        1. 등록한 갤러리 별점 기준
        2. 전시지역
    4. 공고 클릭시 해당 공고 칸이 커지면서 상세 정보와 지원하기 버튼 나옴
        1. 지원하기 버튼은 artist user만 누를 수 있고, 누를 시 마이페이지에서 등록해둔 portfolio가 갤러리 메일로 전송됨
7. 혜택 페이지
    1. 홈화면의 지원 혜택 클릭 / 혹은 Navigation Bar에서 혜택 클릭하면 이동되도록함
    2. admin user가 등록해둔 혜택들이 정리되어 보이도록 함

[구현시 주의 사항]

-개발 환경에서는 로컬서버를 띄워서 확인할 수 있도록 하지만, 추후 배포용으로 상업화 가능성을 두고 수정 가능하도록 구현해야함.

-유지 보수가 가능하도록 간결하면서도 명확한 코드를 작성해야하고 주석 작성이 필요함.

- 유지보수를 위해서 너가 구현하면서 구현한 내용에 대한 사용자 guide를 작성해야함. 사용자 guide에는 개발자를 위해서 해당 내용이 있는 코드 block을 알려주면 좋음. (너무상세하게 모든곳에하면 토큰이 아까우니 큰 틀에서)

- 신규 갤러리 / 공모 / 리뷰 / 추가 및 수정/ 삭제 시에도 DB 및 프론트/백엔드 서버에 전혀 이상이 없이 작동할 수 있도록 해야함

- 단일 기능에 대한 구현 후 검증은 물론이고 여러 기능을 동시 및 순차적으로 실행했을 때에도 문제가 없이 자체검증할 계획을 세우고 검증에 통과할 때까지 개발을 해야함.

- 기능적인 부분은 물론이고 신뢰성 검증에도 중요성을 둬야함.
